    价格监测时效性相关问题排查思路¶

    文档类型: 排查指南 | 更新时间: 2026-06-04 | 维护人: 研发团队 | 标签: 价格监测, 时效性, 排查思路, 租户任务, 全量任务

    排查流程¶

    第一步：明确排查范围¶

    前提条件

    首先需明确哪个租户的什么时间段的商品需要排查

    第二步：获取批次 ID¶

    根据租户 ID 和时间范围，获取推送的租户商品批次 ID：

    -- 获取一定时间范围内推送的租户商品批次id
SELECT distinct batch_id 
FROM t_developer_goods 
WHERE developer_id = '租户id' 
  and operation = 1 
  and modify_time > 'startTime' 
  and modify_time < 'endTime';

    第三步：根据批次 ID 排查¶

    根据批次 ID 可以获取到 t_tenant_task_conf（租户任务配置表）数据，这里有两种情况：

    ===

    #### 情况 1：未查询到 `t_tenant_task_conf` 数据

根据下述 SQL 排查批次是否闭合：

```sql
-- 查询未结束传输的批次
SELECT * 
FROM t_developer_goods_batch 
WHERE (batch_status is null or batch_status = '' or batch_status != '1') 
  and batch_id in ('批次id')
```

??? tip "处理方式"
    如果批次状态正常，则需要手动重新操作一次 `/syncFinish`, 可使用 `InternalManageController` 中提供的 main 方法批量操作。

    ===

    #### 情况 2：`t_tenant_task_conf` 数据存在

查看 `state` 字段：

=== "state = 0（任务已触发）"
    ```sql
    -- 根据 sql 查询是否存在任务表 t_developer_task 数据
    SELECT * 
    FROM t_developer_task 
    WHERE source_id = '租户源id' 
      and batch_id in ('批次id')
    ```

    ??? info "排查说明"
        任务肯定是存在的，主要看 `t_developer_task` 的 `state` 是不是 **1000**（执行成功）。

    ??? bug "如果不是 1000"
        需要立即执行完监测任务的话，可根据 `PriceStartMain` 提供的 main 方法里面的 **直接触发合理价计算** 进行处理，或者通过运营端的监测任务列表页面进行手动操作。

=== "state = 1（任务未触发）"
    ??? bug "可能原因"
        可能堵在打码任务，可以在服务器内根据任务 ID（`tenant_task_id`）搜索日志信息

    ```bash
    # 示例
    grep "2032383091493126145" /boss/softlog/pricemonitor/bssc-biz-operate/maintoperate/info.2026-03-30.0.log
    ```

    ??? example "日志示例（未触发原因：有未核验数据，证明批次内存在未打码商品）"

    ===

    第四步：排查基于时间范围打码的商品数据¶

    可使用下述 SQL 根据时间分组获得大致打码任务触发时间点（时间范围的打码任务 华能是 4 分钟一次，其余是一小时一次）：

    -- `t_tenant_task_conf` 未触发的批次
SELECT 
    count(1) as `数量`,
    DATE_FORMAT(modify_time, '%Y-%m-%d %H%i') AS `时间分组`
FROM t_developer_goods 
WHERE developer_id = '租户id' 
    AND batch_id IN ('批次id')
GROUP BY DATE_FORMAT(modify_time, '%Y-%m-%d %H%i')

    第五步：排查租户批次商品的监测结果¶

    是否传输完毕，可使用 GoodsWithoutHisData 代码进行查询（需要指定租户 source_id 和商品 id）
