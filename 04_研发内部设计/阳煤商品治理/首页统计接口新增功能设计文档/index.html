    首页统计接口新增功能设计文档¶

    文档类型: 接口设计 | 更新时间: 2026年 | 维护人: 研发团队 | 涉及模块: entpur-backend | 标签: 首页统计, 接口设计, entpur-backend, 阳煤, 数据汇总

    一、需求概述¶

    1.1 需求背景¶

    背景说明

    在价格监测平台的首页可视化统计模块中，需要新增两个统计接口，用于展示商品异常上报和价格审核规则的统计数据，帮助运营人员快速了解系统运行状态。

    1.2 需求内容¶

    功能清单

    1. 商品异常上报统计接口：返回异常上报的总数、受理数量和驳回数量
    2. 价格审核规则统计接口：返回触发审核规则为价格正常、异常、预警、无监测结果的商品数量

    二、技术方案设计¶

    2.1 数据库分析¶

    2.1.1 商品异常上报表 (t_developer_goods_exception)¶

    表结构关键字段：

    字段名 说明 备注
    id 主键ID
    tenant_id 租户ID
    goods_id 商品ID
    exception_no 异常编码
    deal_status 处理状态 见下方状态说明
    create_time 创建时间
    deal_time 处理时间

    处理状态说明：

    状态值 说明
    -1 待受理
    0 待处理（已受理）
    1 已解决
    2 未解决
    3 已拒绝（驳回）

    统计逻辑：

    • 总数: 统计指定租户的所有异常上报记录
    • 受理数量: deal_status IN ('0', '1', '2') 的记录数（排除待受理-1和已拒绝-3）
    • 驳回数量: deal_status = '3' 的记录数

    2.1.2 合理价审核日志表 (t_reasonable_price_audit_log)¶

    表结构关键字段：

    字段名 说明 备注
    id 主键ID
    tenant_id 租户ID 需要从关联表获取或通过msg_id/task_id关联
    goods_id 商品ID
    audit_result_type 审核结果类型编码 见下方类型说明
    audit_time 审核时间
    create_time 创建时间

    审核结果类型说明：

    类型编码 说明
    01 价格异常
    02 价格正常
    03 价格预警
    04 无监测结果

    重要注意

    根据现有代码分析，t_reasonable_price_audit_log 表中没有直接的 tenant_id 字段，需要通过以下方式获取租户信息：

    • 方案1：通过 task_id 或 msg_id 关联到任务表获取租户ID
    • 方案2：如果该表实际有 tenant_id 字段但未在实体类中定义，需要确认数据库实际结构

    统计逻辑：

    按 audit_result_type 分组统计各类型的商品数量（去重按 goods_id）

    2.2 接口设计¶

    2.2.1 商品异常上报统计接口¶

    接口信息

    项目 说明
    接口路径 GET /api/monitorStatView/exceptionReportStat
    请求参数 无（从Token中获取tenantId）
    返回格式 JSON
    响应数据结构 
    {
  "code": 0,
  "msg": "success",
  "data": {
    "totalCount": 100,      // 异常上报总数
    "acceptedCount": 80,    // 受理数量
    "rejectedCount": 20     // 驳回数量
  }
}
    响应DTO定义 
    @Data
@SuperBuilder
@NoArgsConstructor
@AllArgsConstructor
@ApiModel(value = "商品异常上报统计响应", description = "商品异常上报统计响应")
public class ExceptionReportStatResponse {
    @ApiModelProperty(value = "异常上报总数")
    private Integer totalCount;

    @ApiModelProperty(value = "受理数量")
    private Integer acceptedCount;

    @ApiModelProperty(value = "驳回数量")
    private Integer rejectedCount;
}

    2.2.2 价格审核规则统计接口¶

    接口信息

    项目 说明
    接口路径 GET /api/monitorStatView/priceAuditStat
    请求参数 无（从Token中获取tenantId）
    返回格式 JSON
    响应数据结构 
    {
  "code": 0,
  "msg": "success",
  "data": {
    "abnormalCount": 50,        // 价格异常数量
    "normalCount": 200,         // 价格正常数量
    "warningCount": 30,         // 价格预警数量
    "noMonitorCount": 10        // 无监测结果数量
  }
}
    响应DTO定义 
    @Data
@SuperBuilder
@NoArgsConstructor
@AllArgsConstructor
@ApiModel(value = "价格审核规则统计响应", description = "价格审核规则统计响应")
public class PriceAuditStatResponse {
    @ApiModelProperty(value = "价格异常数量")
    private Integer abnormalCount;

    @ApiModelProperty(value = "价格正常数量")
    private Integer normalCount;

    @ApiModelProperty(value = "价格预警数量")
    private Integer warningCount;

    @ApiModelProperty(value = "无监测结果数量")
    private Integer noMonitorCount;
}

    2.3 服务层设计¶

    2.3.1 Service接口扩展¶

    修改文件: MonitorStatViewService.java

    在接口中新增两个方法：

    /**
 * 获取商品异常上报统计数据
 *
 * @param tenantId 租户id
 * @return 异常上报统计响应
 */
ExceptionReportStatResponse getExceptionReportStat(String tenantId);

/**
 * 获取价格审核规则统计数据
 *
 * @param tenantId 租户id
 * @return 价格审核规则统计响应
 */
PriceAuditStatResponse getPriceAuditStat(String tenantId);

    2.3.2 Service实现类¶

    修改文件: MonitorStatViewServiceImpl.java

    实现要点：

    1. 注入需要的Mapper：
    2. DeveloperGoodsExceptionMapper（操作 t_developer_goods_exception）

    3. ReasonablePriceAuditLogMapper（操作 t_reasonable_price_audit_log）

    4. 异常上报统计SQL逻辑：

    -- 总数
SELECT COUNT(*) FROM t_developer_goods_exception WHERE tenant_id = #{tenantId}

-- 受理数量（排除待受理-1和已拒绝-3）
SELECT COUNT(*) FROM t_developer_goods_exception 
WHERE tenant_id = #{tenantId} AND deal_status IN ('0', '1', '2')

-- 驳回数量
SELECT COUNT(*) FROM t_developer_goods_exception 
WHERE tenant_id = #{tenantId} AND deal_status = '3'
    1. 价格审核统计SQL逻辑：
    -- 需要先确认tenant_id字段的获取方式
-- 假设通过task_id关联或直接有tenant_id字段

SELECT 
  SUM(CASE WHEN audit_result_type = '01' THEN 1 ELSE 0 END) as abnormalCount,
  SUM(CASE WHEN audit_result_type = '02' THEN 1 ELSE 0 END) as normalCount,
  SUM(CASE WHEN audit_result_type = '03' THEN 1 ELSE 0 END) as warningCount,
  SUM(CASE WHEN audit_result_type = '04' THEN 1 ELSE 0 END) as noMonitorCount
FROM t_reasonable_price_audit_log
WHERE tenant_id = #{tenantId}

    注意事项

    • 需要对 goods_id 进行去重统计，避免同一商品多次审核被重复计算
    • 可以考虑只统计最近一段时间的审核记录（如最近30天），或者统计所有历史记录

    2.4 Controller层设计¶

    修改文件: MonitorStatViewController.java

    新增两个接口方法：

    @GetMapping("/exceptionReportStat")
@ApiOperation(value = "商品异常上报统计")
public ResultBody exceptionReportStat() {
    TokenInfo tokenInfo = TokenInfoContext.getTokenInfo();
    String tenantId = tokenInfo.getTenantId();
    if (StringUtils.isEmpty(tenantId)) {
        return ResultBody.failed().msg("未获取到租户信息");
    }
    ExceptionReportStatResponse response = monitorstatViewService.getExceptionReportStat(tenantId);
    return ResultBody.ok().data(response);
}

@GetMapping("/priceAuditStat")
@ApiOperation(value = "价格审核规则统计")
public ResultBody priceAuditStat() {
    TokenInfo tokenInfo = TokenInfoContext.getTokenInfo();
    String tenantId = tokenInfo.getTenantId();
    if (StringUtils.isEmpty(tenantId)) {
        return ResultBody.failed().msg("未获取到租户信息");
    }
    PriceAuditStatResponse response = monitorstatViewService.getPriceAuditStat(tenantId);
    return ResultBody.ok().data(response);
}

    2.5 Mapper层设计¶

    2.5.1 DeveloperGoodsExceptionMapper¶

    修改文件: DeveloperGoodsExceptionMapper.java

    方法1: 使用注解方式 
    /**
 * 统计异常上报总数
 */
@Select("SELECT COUNT(*) FROM t_developer_goods_exception WHERE tenant_id = #{tenantId}")
Integer countExceptionTotal(@Param("tenantId") String tenantId);

/**
 * 统计受理数量
 */
@Select("SELECT COUNT(*) FROM t_developer_goods_exception WHERE tenant_id = #{tenantId} AND deal_status IN ('0', '1', '2')")

/**
 * 统计驳回数量
 */
@Select("SELECT COUNT(*) FROM t_developer_goods_exception WHERE tenant_id = #{tenantId} AND deal_status = '3'")

    推荐方式: 使用XML配置（便于维护） 
    <!-- 统计异常上报数据 -->
<select id="countExceptionStat" resultType="map">
    SELECT 
        COUNT(*) as totalCount,
        SUM(CASE WHEN deal_status IN ('0', '1', '2') THEN 1 ELSE 0 END) as acceptedCount,
        SUM(CASE WHEN deal_status = '3' THEN 1 ELSE 0 END) as rejectedCount
    FROM t_developer_goods_exception
    WHERE tenant_id = #{tenantId}
</select>

    2.5.2 ReasonablePriceAuditLogMapper¶

    修改文件: ReasonablePriceAuditLogMapper.java

    XML配置方式 
    <!-- 统计价格审核结果 -->
<select id="countPriceAuditStat" resultType="map">
    SELECT 
        SUM(CASE WHEN audit_result_type = '01' THEN 1 ELSE 0 END) as abnormalCount,
        SUM(CASE WHEN audit_result_type = '02' THEN 1 ELSE 0 END) as normalCount,
        SUM(CASE WHEN audit_result_type = '03' THEN 1 ELSE 0 END) as warningCount,
        SUM(CASE WHEN audit_result_type = '04' THEN 1 ELSE 0 END) as noMonitorCount
    FROM t_reasonable_price_audit_log
    WHERE tenant_id = #{tenantId}
</select>

    重要提示

    需要确认 t_reasonable_price_audit_log 表是否有 tenant_id 字段。如果没有，需要通过JOIN查询获取：

    SELECT 
    SUM(CASE WHEN l.audit_result_type = '01' THEN 1 ELSE 0 END) as abnormalCount,
    SUM(CASE WHEN l.audit_result_type = '02' THEN 1 ELSE 0 END) as normalCount,
    SUM(CASE WHEN l.audit_result_type = '03' THEN 1 ELSE 0 END) as warningCount,
    SUM(CASE WHEN l.audit_result_type = '04' THEN 1 ELSE 0 END) as noMonitorCount
FROM t_reasonable_price_audit_log l
INNER JOIN t_monitor_task_record t ON l.task_id = t.operate_task_id
WHERE t.tenant_id = #{tenantId}

    4.1 技术风险¶

    风险项 风险等级 应对措施
    t_reasonable_price_audit_log表缺少tenant_id字段 中 提前确认表结构，准备JOIN查询方案
    大数据量下查询性能问题 中 添加合适的索引，考虑分页或缓存
    统计数据准确性问题 低 充分测试，与业务方确认统计口径
