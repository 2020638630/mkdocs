## :material-information: 概述

### 核心功能

本次编码实现了华能消费帮扶商品的独立租户路由功能，通过配置化的方式将消费帮扶商品（goodsType="99"）从华能租户（1706958926295019522）自动路由到华能消费帮扶租户（2018950843406299138），实现数据完全隔离。

**核心思路**: 采用租户→商品类型映射关系表，动态获取目标租户ID，避免硬编码，提高系统扩展性。

**核心代码**: TenantGoodsTypeMappingService 处理租户分配业务，基于新增的租户→商品类型映射关系表获取对应租户 id。为了避免模块依赖问题，当前代码分别在需要的模块中创建了独立的服务实现，涉及服务模块：`bssc-biz-operate`、`bssc-biz-interchange`。

**数据源编码**: 
- 原数据源拼接后缀 `-xfbf`（测试账号数据源：`hncs-xfbf-zncszh`）

**AI端数据源编码**: `hnzhgyl-xfbf` 

**租户id**:

```java
     * 创建人

     * 修改人

    private String updateUser;

     * 是否删除：0=否，1=是
     */
    @TableField("is_delete")
    private Integer isDelete;
    
    /**
     * 逻辑删除标志
     */
    @TableLogic
    private Integer deleted;
    
    /**
     * 商品类型枚举
     */
    public enum GoodsType {
        CONSUMPTION_ASSISTANCE("99", "消费帮扶商品");

        private String name;

            this.value = value;
            this.name = name;

            for (GoodsType type : GoodsType.values()) {
```

```java
    @Override
    public String getTargetTenantId(String originalTenantId, String goodsType) {
        if (originalTenantId == null || goodsType == null) {
            log.info("租户ID或商品类型为空，返回原始租户ID");
            return originalTenantId;
        }

        TenantGoodsTypeMappingEntity mapping = mappingMapper.selectTargetTenant(originalTenantId, goodsType);

        if (mapping != null) {
            log.info("租户映射：{} + {} → {}", originalTenantId, goodsType, mapping.getTargetTenantId());
            return mapping.getTargetTenantId();
        }

        // 没有映射规则则返回原租户ID
        log.info("租户 {} 的商品类型 {} 无映射规则，使用原始租户", originalTenantId, goodsType);
        return originalTenantId;
    }

        // 是否华能租户
        log.info("检查租户 {} 是否为华能租户", tenantId);
        if("1706958926295019522".equals(tenantId) || "1798558397867671554".equals(tenantId)
                || "1706958926295019523".equals(tenantId) ) {
            return true;
        }else{
            return false;
        }
    }

    @Override
    public boolean saveMapping(TenantGoodsTypeMappingEntity mapping, String createUser) {
        try {
            // 检查是否已存在相同映射
            if (existsMapping(mapping.getOriginalTenantId(), mapping.getGoodsType())) {
                log.info("租户 {} 的商品类型 {} 映射已存在", mapping.getOriginalTenantId(), mapping.getGoodsType());
                return false;
            }
            
            // 设置创建信息
            mapping.setCreateTime(LocalDateTime.now());
            mapping.setUpdateTime(LocalDateTime.now());
            mapping.setCreateUser(createUser);
            mapping.setUpdateUser(createUser);

            // 枚举设置商品类型名称
            mapping.setGoodsTypeName(TenantGoodsTypeMappingEntity.GoodsType.getGoodsTypeName(mapping.getGoodsType()));

            mapping.setIsDelete(0);
            mapping.setDeleted(0);

            if (mapping.getStatus() == null) {
                mapping.setStatus(1); // 默认启用
            }
            
            int result = mappingMapper.insert(mapping);
            if (result > 0) {
                log.info("新增租户映射：{}({}) → {}", mapping.getOriginalTenantId(), mapping.getGoodsType(), mapping.getTargetTenantId());
                return true;
            }
            
            return false;
        } catch (Exception e) {
            log.error("保存租户映射失败: {}", e.getMessage(), e);
            return false;
        }
    }

            mapping.setUpdateUser(updateUser);

            if (result > 0) {
                log.info("更新租户映射：ID={}, {}({}) → {}",

                return true;
            }
            
            return false;
        } catch (Exception e) {
            log.error("更新租户映射失败", e);
            return false;
        }
    }

            mapping.setId(id);

        } catch (Exception e) {
```

```java
    public ResultBody synchronizeGoods(@RequestBody @Valid DevReq<GoodsSyncReq> req) {
        if (req.getData().getGoods().size() > 100) {
            return ResultBody.failed().msg("一次最多同步100条商品数据");
        }
        TokenInfo tokenInfo = TokenInfoContext.getTokenInfo();

        // lilinlin 华能消费帮扶需求
        String originalTenantId = tokenInfo.getTenantId();
        log.info("商品同步开始，租户ID：{}", originalTenantId);
        // 检查是否为华能租户
        if (tenantGoodsTypeMappingService.isHuaNengTenantId(originalTenantId)) {

            List<GoodsEntity> allGoods = req.getData().getGoods();
            if (CollectionUtils.isEmpty(allGoods)) {
                return ResultBody.failed().msg("商品不存在");
            }

            // 按goodsType分组 key
            List<String> goodsTypeList = allGoods.stream().map(GoodsEntity::getGoodsType).distinct().collect(Collectors.toList());
            if (goodsTypeList.size() > 1) {
                return ResultBody.failed().msg("商品同步请求中包含多个goodsType，请检查");
            }
            String goodsType = allGoods.get(0).getGoodsType();
            log.info("当前商品类型 goodsType: {}", goodsType);
            String targetTenantId = tenantGoodsTypeMappingService.getTargetTenantId(originalTenantId, goodsType);
            log.info("根据商品类型获取到的租户id:{}",targetTenantId);
            // 使用目标租户ID
            tokenInfo.setTenantId(targetTenantId);
            tokenInfo.setEnterpriseId(targetTenantId);
        }
        // 其他租户保持原逻辑
        boolean isPrivateTenant = interChangeUtils.isPrivateTenant(originalTenantId);
        if(isPrivateTenant) {
            return tDeveloperGoodsService.synchronizeGoods(req, tokenInfo);
        } else {
            return tDeveloperGoodsService.synchronizeBatchGoods(req, tokenInfo);
        }
    }
```

## :material-folder-sync: 华能双租户品目处理

使用下述两个方法，如果为华能租户，则分别设置源租户与映射租户品目缓存

-- 对应 20. 首页配置中心 -- 监测品目导入

```java
@Override
    public void setTenantCatalogsCache(RedisTemplate redisTemplate, String tenantId, List<? extends Object> tenantCatalogsList) {
        if (redisTemplate == null || tenantId == null || tenantCatalogsList == null) {
            log.error("设置租户品目缓存参数为空");
            return;
        }
        // lilinlin 华能消费帮扶需求，设置源租户缓存
        String key = CacheKey.TENANT_CATALOGS + tenantId;
        redisTemplate.opsForList().leftPushAll(key, tenantCatalogsList);
        log.info("设置租户 {} 品目缓存成功", tenantId);

        // lilinlin 华能消费帮扶需求，为虚拟租户设置相同缓存
        if (isHuaNengTenantId(tenantId)) {
            List<TenantGoodsTypeMappingEntity> mappings = getMappingsByOriginalTenantId(tenantId);
            if (mappings != null && !mappings.isEmpty()) {
                for (TenantGoodsTypeMappingEntity mapping : mappings) {
                    String targetTenantId = mapping.getTargetTenantId();
                    String targetKey = CacheKey.TENANT_CATALOGS + targetTenantId;
                    redisTemplate.opsForList().leftPushAll(targetKey, tenantCatalogsList);
                    log.info("设置虚拟租户 {} 品目缓存成功（来源租户 {}）", targetTenantId, tenantId);
                }
            }
        }
    }

    @Override
    public void deleteTenantCatalogsCache(RedisTemplate redisTemplate, String tenantId) {
        if (redisTemplate == null || tenantId == null) {
            log.error("删除租户品目缓存参数为空");
            return;
        }
        // lilinlin 华能消费帮扶需求，删除源租户缓存
        String key = CacheKey.TENANT_CATALOGS + tenantId;
        redisTemplate.delete(key);
        log.info("删除租户 {} 品目缓存成功", tenantId);

        // lilinlin 华能消费帮扶需求，删除虚拟租户缓存
        if (isHuaNengTenantId(tenantId)) {
            List<TenantGoodsTypeMappingEntity> mappings = getMappingsByOriginalTenantId(tenantId);
            if (mappings != null && !mappings.isEmpty()) {
                for (TenantGoodsTypeMappingEntity mapping : mappings) {
                    String targetTenantId = mapping.getTargetTenantId();
                    String targetKey = CacheKey.TENANT_CATALOGS + targetTenantId;
                    redisTemplate.delete(targetKey);
                    log.info("删除虚拟租户 {} 品目缓存成功（来源租户 {}）", targetTenantId, tenantId);
                }
            }
        }
    }
```

## :material-architecture: 技术架构

### 核心设计模式

1. **策略模式**: 通过租户商品类型映射表实现动态路由策略
2. **工厂模式**: TenantGoodsTypeMappingService 作为租户路由工厂
3. **缓存模式**: 双租户缓存同步机制

### 关键技术点

1. **租户隔离**: 通过商品类型自动路由到不同租户
2. **数据同步**: 品目、品牌、供应商等基础数据双租户同步
3. **缓存一致性**: 保证两个租户的缓存数据一致
4. **向后兼容**: 不影响其他租户的正常使用

## :material-note: 注意事项

1. **数据源命名规范**: 消费帮扶数据源统一使用 `-xfbf` 后缀
2. **租户ID 配置**: 生产环境和测试环境租户ID 不同，需注意区分
3. **缓存清理**: 修改配置后需清理 Redis 缓存 `APP:TENANT:DEVELOPER:`
4. **枚举更新**: 新增租户需在 TenantMessageEnum 中注册
5. **权限控制**: 确保两个租户的权限相互独立
