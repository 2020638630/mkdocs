## 概述

### 背景

华能消费帮扶商品需要与普通商品实现数据隔离，通过配置化的方式将消费帮扶商品（`goodsType="99"`）从华能主租户自动路由到华能消费帮扶独立租户。

### 核心思路

采用 **租户→商品类型映射关系表**（`t_tenant_goods_type_mapping`），动态获取目标租户ID，避免硬编码，提高系统扩展性。

### 租户信息

| 环境 | 租户ID | Code | 名称 |
|------|--------|------|------|
| 生产 | `1706958926295019522` | `hnzhgyl` | 华能智链（主租户） |
| 生产 | `2018950843406299138` | `hnzhgyl-xfbf` | 华能智链消费帮扶 |
| 测试 | `1798558397867671554` | `hnzhgyl` | 华能测试（主租户） |
| 测试 | `2019308420509958145` | `hnzhgyl-xfbf` | 华能测试消费帮扶 |

### 数据源命名规范

消费帮扶数据源统一使用 `-xfbf` 后缀：
- 原数据源编码拼接后缀 `-xfbf`（如：`hncs-xfbf-zncszh`）
- AI端数据源编码：`hnzhgyl-xfbf`

## 数据库表结构

### t_tenant_goods_type_mapping（租户商品类型映射表）

```sql
CREATE TABLE `t_tenant_goods_type_mapping` (
  `id` bigint(20) NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `original_tenant_id` varchar(64) NOT NULL COMMENT '原始租户ID',
  `target_tenant_id` varchar(64) NOT NULL COMMENT '目标租户ID',
  `goods_type` varchar(10) NOT NULL COMMENT '商品类型：99=消费帮扶商品',
  `goods_type_name` varchar(255) DEFAULT NULL COMMENT '商品类型名称',
  `description` varchar(200) DEFAULT NULL COMMENT '映射描述',
  `status` tinyint(2) NOT NULL DEFAULT '1' COMMENT '状态：1=启用，0=禁用',
  `create_time` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `update_time` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  `create_user` varchar(64) DEFAULT NULL,
  `update_user` varchar(64) DEFAULT NULL,
  `is_delete` tinyint(2) NOT NULL DEFAULT '0' COMMENT '是否删除：0=否，1=是',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_tenant_goods_type` (`original_tenant_id`,`goods_type`,`is_delete`),
  KEY `idx_original_tenant` (`original_tenant_id`),
  KEY `idx_goods_type` (`goods_type`),
  KEY `idx_status` (`status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='租户商品类型映射表';
```

## 核心服务设计

### TenantGoodsTypeMappingService

核心服务类，位于 `bssc-biz-operate` 和 `bssc-biz-interchange` 模块（为避免模块依赖，各模块独立实现）。

**关键方法**：

| 方法 | 说明 |
|------|------|
| `getTargetTenantId(originalTenantId, goodsType)` | 根据原租户ID + 商品类型查映射表，返回目标租户ID；无映射返回原租户ID |
| `isHuaNengTenantId(tenantId)` | 判断是否为华能租户（含生产/测试环境所有华能租户ID） |

**查询逻辑**（Mapper SQL）：

```sql
SELECT * FROM t_tenant_goods_type_mapping
WHERE original_tenant_id = #{originalTenantId}
  AND goods_type = #{goodsType}
  AND status = 1
  AND is_delete = 0
ORDER BY priority ASC, create_time ASC
LIMIT 1
```

## 各模块修改策略

### 1. 商品同步（bssc-biz-interchange）

**策略**：在 `/sync` 和 `/syncFinish` 接口中，华能租户根据商品类型重新设置 TokenInfo 中的 tenantId，后续流程自动使用目标租户。

### 2. 商品导入（bssc-biz-entpur）

**策略**：三个导入页面（普通导入、归一导入、下架导入）均新增 `goodsType` 参数。华能租户根据 goodsType 重新设置目标租户ID后执行导入。

### 3. 商品列表查询（bssc-biz-entpur → bssc-biz-operate）

**策略**：华能租户查询时，同时查主租户 + 所有映射租户（`tenantIdList`），QueryOriginGoodsRequest 新增 `goodsType`、`tenantIdList` 条件，返回体 TenantGoodsBO 新增 `developerId`。

### 4. 品目/品牌/供应商同步（bssc-biz-interchange）

**策略**：华能租户同步时，分别对源租户和目标租户执行两次保存操作，并对两个租户的 Redis 缓存进行处理。

### 5. 品目缓存管理

华能租户设置/删除品目缓存时，同时处理源租户和所有映射租户的缓存：
- `setTenantCatalogsCache` — 同时为映射租户设置相同缓存
- `deleteTenantCatalogsCache` — 同时删除映射租户的缓存

### 6. 消息接口（bssc-biz-interchange）

**策略**：
- **消息配置保存**（`/confirm`）：华能租户同时给两个租户各存一套配置
- **消息查询**（`/getMsgData`）：华能租户使用 `tenantId IN (...)` 查询
- **消息删除**（`/msgDel`）：同理使用 `IN` 查询过滤

### 7. 合理价查询（bssc-biz-interchange）

**策略**：华能租户分别查询主租户和帮扶租户的合理价数据，聚合返回。

### 8. 监测任务记录（bssc-biz-interchange）

**策略**：`/listForApi` 接口，华能租户返回所有关联租户的任务数据（`tenantIdList`）。

### 9. 定时任务（bssc-biz-operate）

**策略**：
- `/collectAndCodeTaskByTime`：使用 `isHuaNengTenantId` 替换硬编码判断
- `/buildPriceMonitorTask`：排除华能数据源时也排除 `xfbf-` 前缀的关联数据源
- `/buildHnzhgylPriceMonitorTask`（华能专用）：分别对主数据源和 `xfbf-` 前缀数据源构建监测任务

### 10. AI端枚举（enterprise_acquisition）

在 `TenantMessageEnum.java` 中补充华能消费帮扶租户枚举值：

```java
// 测试环境
TENANT_2019308420509958145(2019308420509958145L, "hnzhgyl-xfbf", "华能测试消费帮扶"),
// 正式环境
TENANT_2018950843406299138(2018950843406299138L, "hnzhgyl-xfbf", "华能智链消费帮扶"),
```

## 通用处理模式

各接口修改遵循统一的华能租户处理模式：

```
1. 判断是否为华能租户 → isHuaNengTenantId(tenantId)
2. 获取关联租户列表 → getMappingsByOriginalTenantId(tenantId)
3. 数据查询/写入 → 使用多租户 IN 查询 或 双写
```

### 多租户查询 SQL 模式

```sql
-- 原SQL
WHERE tenant_id = #{tenantId}

-- 修改后
<if test="tenantIdList != null and tenantIdList.size() > 0">
    AND tenant_id IN
    <foreach collection="tenantIdList" item="item" open="(" separator="," close=")">
        #{item}
    </foreach>
</if>
<if test="tenantIdList == null or tenantIdList.size() == 0">
    AND tenant_id = #{tenantId}
</if>
```

### 双写模式

品目/品牌/供应商同步等场景，构造目标租户的 TokenInfo 后分别调同一方法写入：

```java
// 源租户写入
service.saveOrUpdate(req, tokenInfo);
// 目标租户写入
TokenInfo targetTokenInfo = new TokenInfo();
targetTokenInfo.copyFrom(tokenInfo);
targetTokenInfo.setTenantId(targetTenantId);
service.saveOrUpdate(req, targetTokenInfo);
```
